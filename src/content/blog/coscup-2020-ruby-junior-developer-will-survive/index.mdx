吃寫程式的頭路大概一年九個月了，這段時間學了很多東西、看了很多的書，很多觀念和資訊在我們公司以及在 ASTRO Camp 當助教的時候其實都算是老生常談了。但是每次經手到外面的程式碼常常會發現「天啊～這不是常識嗎？怎麼會這樣寫」，才意識到不是所有的地方都是依照正確的觀念在執行，所以今年的 Coscup (2020) 決定[投稿一篇](https://coscup.org/2020/zh-TW/agenda/QW8PXE)關於這段時間我在開發 Ruby on Rails 這段期間學到的以及看到覺得不錯的書、文章、影片。

第一部份是關於需求溝通這塊。畢竟活了這把年紀，以前都是靠嘴砲騙吃騙喝的，但是要拿這個「如何嘴砲」當作主題來發表真的難到靠北，為了真的不要讓人家覺得我隨便講，花了很多時間看了一些麥肯錫的書，結論是邏輯訓練不是看了什麼金字塔就會的，需要多花一點時間練習。後面就是講有關一些中階 Rails 工程師應該要知道的東西，另外就是 clean code 的重要性，算是專案上的血淚分享。

因為想把不錯的資源都放進去，30 分鐘的演講我弄了 200 多頁投影片，果然很多都講不完 XDD，很多橋段精心準備迷因都沒辦法好好的給大家看覺得有點可惜，但是我的目的還是希望這份投影片可以幫助很多人或很多公司，他們在迷惘或者是公司前輩懶得教人的時候可以拿這份投影片去參考!

另外平常在 ASTRO Camp 在那邊叼學員 Demoday 發表的簡報不好怎樣的，真的自己上台簡報才發現有夠困難啊....
